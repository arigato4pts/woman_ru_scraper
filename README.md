## Быстрый старт

```bash
# Собрать 1 000 постов (настройки по умолчанию из config.py)
python scraper.py

# Собрать 500 постов из раздела «Отношения»
python scraper.py --limit 500 --sections relations

# Собрать 5 000 постов из нескольких разделов
python scraper.py --limit 5000 --sections relations family psychology

# Собрать все посты подряд без фильтрации по ключевому слову
python scraper.py --limit 2000 --no-filter

# Указать свою папку и имя файла
python scraper.py --limit 1000 --output-dir data/run2 --filename corpus_march
```

---

## Параметры командной строки

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `--limit N` | int | 1000 | Максимальное число постов для сохранения |
| `--sections S1 S2 …` | str+ | из config.py | Слаги разделов форума |
| `--no-filter` | flag | выкл | Отключить фильтрацию по слову «мужчина» |
| `--output-dir PATH` | str | `data/raw` | Папка для сохранения результатов |
| `--filename NAME` | str | `woman_ru_muzchina` | Имя файла без расширения |

### Доступные разделы (`--sections`)

| Слаг | Раздел форума |
|------|--------------|
| `relations` | Отношения |
| `family` | Семья |
| `psychology` | Психология |
| `kids` | Дети |

> Список можно расширить в `config.py → FORUM_SECTIONS`.

## Настройка через config.py

Файл `config.py` — единая точка управления всеми параметрами.  
Менять код `scraper.py` не нужно.

### Ключевые параметры

```python
# Лимиты
DEFAULT_POST_LIMIT = 1000       # постов по умолчанию
MAX_THREADS_PER_SECTION = 200   # тредов на раздел
MAX_PAGES_PER_THREAD = 10       # страниц пагинации в треде

# Скорость (вежливость к серверу)
REQUEST_DELAY_MIN = 1.5   # сек минимум между запросами
REQUEST_DELAY_MAX = 3.5   # сек максимум между запросами
MAX_RETRIES = 3           # попыток при ошибке сети

# Форматы сохранения
OUTPUT_FORMATS = ["csv", "json", "txt"]   # можно убрать ненужные

# Целевые словоформы (расширяемый список)
TARGET_WORDFORMS = ["мужчина", "мужчины", "мужчине", ...]
```

## Формат выходных данных

Каждый пост сохраняется в трёх форматах одновременно.

### CSV (`data/raw/*.csv`)

```
post_id,thread_id,thread_title,section,post_date,post_text,url
a1b2c3d4e5f6,123456,Почему мужчины не звонят?,relations,2024-03-15,...
```

Кодировка: **UTF-8 BOM** (корректно открывается в Excel).

### JSON (`data/raw/*.json`)

```json
[
  {
    "post_id": "a1b2c3d4e5f6",
    "thread_id": "123456",
    "thread_title": "Почему мужчины не звонят?",
    "section": "relations",
    "post_date": "2024-03-15T10:23:00",
    "post_text": "Мой мужчина совсем перестал писать...",
    "url": "https://www.woman.ru/forum/relations/123456/"
  }
]
```

### TXT (`data/raw/*.txt`)

Каждый пост разделён горизонтальной линией, содержит метаданные  
и полный текст — удобно для ручной проверки выборки.

## Алгоритм работы

```
scraper.py
│
├─ parse_args()              — читает параметры CLI
│
└─ run_scraper()
   ├─ for section in sections:
   │  └─ iter_thread_urls()  — пагинация по списку тредов в разделе
   │     └─ for thread_url:
   │        └─ iter_posts_in_thread()   — пагинация внутри треда
   │           └─ for post:
   │              ├─ дедупликация (MD5 по тексту + дате)
   │              ├─ contains_target()  — проверка словоформ
   │              └─ append → collected[]
   │
   └─ save_all()             — CSV + JSON + TXT
```

**Вежливый режим:** между каждым запросом случайная пауза  
`REQUEST_DELAY_MIN … REQUEST_DELAY_MAX` секунд.  
При ошибке сети — экспоненциальный backoff (2, 4, 8 сек).

## Этические и правовые аспекты

- Скрипт используется **исключительно в учебных целях**.
- Между запросами выдерживается задержка — нагрузка на сервер минимальна.
- Собираются только общедоступные тексты с публичного форума.
- Личные данные пользователей (имена, аватары, e-mail) **не собираются**.
- Данные хранятся локально и не публикуются в открытом доступе.
- Перед масштабным сбором рекомендуется проверить  
  `https://www.woman.ru/robots.txt` и условия использования сайта.
